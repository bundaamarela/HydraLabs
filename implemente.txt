MODELO DE DADOS — adiciona ao schema Prisma:

model Workspace {
  id          String   @id @default(cuid())
  title       String
  description String?
  projectId   String?
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
  project     Project? @relation(fields:[projectId],
                         references:[id])
  blocks      Block[]
}

model Block {
  id          String    @id @default(cuid())
  workspaceId String
  type        String    -- 'text' | 'heading' | 
                        -- 'insight' | 'quote' | 
                        -- 'divider' | 'session-ref'
  content     String?
  meta        String?   -- JSON com dados extras
  order       Int
  createdAt   DateTime  @default(now())
  workspace   Workspace @relation(fields:[workspaceId],
                          references:[id],
                          onDelete:Cascade)
}

Corre: npx prisma db push

ROTAS API:

/app/api/workspaces/route.ts
  GET  → lista workspaces (com contagem de blocos)
  POST → cria { title, description, projectId }

/app/api/workspaces/[id]/route.ts
  GET    → workspace completo com todos os blocos
  PATCH  → actualiza title ou description
  DELETE → apaga workspace e blocos

/app/api/workspaces/[id]/blocks/route.ts
  GET    → lista blocos ordenados por order
  POST   → cria bloco { type, content, meta, order }
  PATCH  → reordena blocos { blocks: [{id, order}] }

/app/api/workspaces/[id]/blocks/[blockId]/route.ts
  PATCH  → actualiza content ou meta
  DELETE → apaga bloco
